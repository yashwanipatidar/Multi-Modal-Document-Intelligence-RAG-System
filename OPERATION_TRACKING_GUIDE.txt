DETAILED OPERATION TRACKING - IMPLEMENTATION GUIDE

This guide explains how detailed operation tracking has been integrated into the RAG system.

===========================================
1. OPERATION TRACKING MODULE
===========================================

File: src/operation_tracker.py

Provides:
- OperationTracker: Main class for tracking operations
- OperationStatus: Enum for operation states (PENDING, IN_PROGRESS, SUCCESS, FAILED, WARNING)
- OperationLog: Data class for storing operation details
- OperationTimer: Context manager for automatic timing
- get_operation_tracker(): Factory function

Key Features:
✅ Automatic timing with start/end timestamps
✅ Status tracking (success/failure/warnings)
✅ Detailed operation metrics
✅ File and console logging
✅ Session-based log organization
✅ JSON-based log files for analysis


===========================================
2. TRACKED OPERATIONS
===========================================

The following operations are now tracked in demo_app.py:

A. PDF UPLOAD
   Operation: "PDF Upload"
   Tracks:
   - Number of files uploaded
   - File sizes (in MB)
   - File paths
   - Duration of upload process
   - Success/failure status

B. TEXT & IMAGE EXTRACTION
   Operation: "PDF Text & Image Extraction"
   Tracks:
   - PDF count
   - Number of chunks extracted
   - Duration
   - OCR status
   - Image extraction details

C. TABLE EXTRACTION
   Operation: "Table Extraction"
   Tracks:
   - PDF count
   - Number of tables found
   - Duration
   - Extraction success rate

D. MULTI-MODAL INDEX BUILDING
   Operation: "Multi-Modal Index Building"
   Tracks:
   - Number of chunks processed
   - Number of tables indexed
   - Index file size (MB)
   - Duration of entire indexing process
   - Memory usage

E. QUERY RETRIEVAL (UNIFIED MODE)
   Operation: "Query Retrieval (Unified)"
   Tracks:
   - Query text (first 100 chars)
   - Top-k parameter used
   - Results found count
   - Retrieval time (seconds)
   - Modality composition of results

F. QUERY RETRIEVAL (BY MODALITY)
   Operation: "Query Retrieval (By Modality)"
   Tracks:
   - Query text (first 100 chars)
   - Top-k parameter
   - Results per modality (text/image/table)
   - Total retrieval time
   - Modality distribution


===========================================
3. HOW TO USE
===========================================

A. AUTOMATIC TRACKING (Built-in)
   Operations are tracked automatically. No additional code needed.

B. ADDING DETAILS DURING OPERATION
   from src.operation_tracker import OperationTimer
   
   tracker = _get_or_create_tracker()
   with OperationTimer(tracker, "My Operation", {"param1": "value1"}):
       # Your code here
       tracker.add_detail("metric", 123)
       tracker.add_warning("Something to note")

C. ACCESSING LOGS
   tracker = _get_or_create_tracker()
   
   # Get summary
   summary = tracker.get_operations_summary()
   print(summary)
   # {
   #     "total_operations": 5,
   #     "total_duration": 45.32,
   #     "success_count": 4,
   #     "failed_count": 1,
   #     "success_rate": 80.0
   # }
   
   # Get detailed logs
   details = tracker.get_operation_details("PDF Upload")
   print(details)


===========================================
4. LOG OUTPUT LOCATIONS
===========================================

A. STREAMLIT UI (In Sidebar)
   📋 Operation Logs Section shows:
   - Operation metrics (total ops, success rate, total time)
   - Detailed expandable logs for each operation
   - Status indicators (✅ SUCCESS, ❌ FAILED, ⚠️ WARNING)
   - Error messages and warnings
   - Detailed parameters for each operation

B. FILE SYSTEM
   Location: data/processed/session_logs/
   
   Files created per session:
   - operations_{session_id}.jsonl
     └─ One JSON line per operation (for log analysis)
   - debug_{session_id}.log
     └─ Full debug logs with timestamps

   Example structure:
   data/processed/session_logs/
   ├── operations_abc123xyz.jsonl
   ├── debug_abc123xyz.log
   ├── operations_def456uvw.jsonl
   ├── debug_def456uvw.log
   └── ...

   import json
   from pathlib import Path
   
   session_log = Path("data/processed/session_logs/operations_xyz.jsonl")
   
   for line in session_log.read_text().strip().split('\n'):
       op = json.loads(line)
       print(f"{op['operation_name']}: {op['duration_seconds']}s - {op['status']}")

C. PERFORMANCE METRICS
   tracker = _get_or_create_tracker()
   
   # Get all extract operations
   extract_ops = tracker.get_operation_details("PDF Text & Image Extraction")
   
   # Calculate average duration
   avg_duration = sum(op['duration_seconds'] for op in extract_ops) / len(extract_ops)
   print(f"Average extraction time: {avg_duration:.2f}s")
